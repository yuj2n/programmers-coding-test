# [Programmers] 가장 큰 수 (Level 2)

핵심 키워드: `#Sort` `#String_Concatenation` `#Greedy`

---

## 1. 문제 설명

주어진 정수 배열을 조합하여 만들 수 있는 **가장 큰 수**를 찾는 문제.

- **핵심 포인트**: 숫자의 단순 크기가 아니라, 이어 붙였을 때 어떤 숫자가 앞에 와야 더 큰지 결정해야 함!

---

## 2. 풀이 방식 (Custom Sorting 활용)

단순한 내림차순 정렬이 아닌, **문자열 결합 결과를 비교하는 사용자 정의 정렬**을 사용.

### 💡 로직 설명

1. **문자열 변환**: 숫자를 이어 붙이기 위해 `map(String)`을 사용하여 문자열 배열로 변환합니다.
2. **조합 비교 정렬**: 두 수 `a`, `b`에 대해 `(b + a)`와 `(a + b)`의 크기를 비교합니다.
   - 예: `10`과 `2` 비교 시, `"210"` vs `"102"` → `"210"`이 크므로 `2`가 앞으로 옵니다.
3. **결과 결합**: 정렬된 배열을 `join('')`으로 합칩니다.
4. **예외 처리**: 모든 숫자가 `0`인 경우 `"000..."`이 아닌 `"0"`을 반환하도록 처리합니다.

```javascript
function solution(numbers) {
  const answer = numbers
    .map(String)
    .sort((a, b) => b + a - (a + b))
    .join("");

  return answer[0] === "0" ? "0" : answer;
}
```

## 3. 시행착오 및 해결 과정

### ❌ 일반 정렬의 문제

- `[3, 30]`의 경우 일반 내림차순은 `30, 3` 순이지만, 조합 시 `303`보다 `330`이 크기 때문에 일반 정렬로는 해결할 수 없었음.

### ✅ 정렬 조건 최적화

- **문자열 연산 활용**: `(b + a) - (a + b)` 수식을 통해 두 조합의 우선순위를 한 줄로 판별했음.
- **방어 코드**: `numbers`가 `[0, 0]`일 때의 반환값을 체크하여 엣지 케이스를 해결함.
