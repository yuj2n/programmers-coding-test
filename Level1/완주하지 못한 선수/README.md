# [Programmers] 완주하지 못한 선수 (Level 1)

## 1. 문제 설명

마라톤 참가자 중 완주하지 못한 단 한 명의 선수를 찾는 문제입니다. 동명이인이 존재할 수 있다는 점이 핵심입니다.

---

## 2. 풀이 방식 (Hash Map 활용)

기존의 정렬(Sort) 방식에서 **Hash Map(JavaScript Map 객체)**을 활용한 방식으로 리팩토링을 진행했습니다.

### 💡 로직 설명

1. **Map 객체 생성**: 참가자 이름을 Key로, 해당 이름의 인원수를 Value로 저장하는 `pMap`을 생성합니다.
2. **해시 카운팅**: `participant` 배열을 순회하며 이름을 등록합니다. 이때 동명이인이 있다면 인원수(Value)가 올라갑니다.
3. **완주자 대조**: `completion` 배열을 순회하며 완주한 사람의 인원수를 1씩 차감합니다.
4. **결과 도출**: 최종적으로 인원수가 1인 Key를 찾아 반환합니다.

### 🚀 성능 비교

- **정렬 방식**: $O(N \log N)$ - 배열을 정렬하는 데 비용이 발생합니다.
- **해시 방식**: $O(N)$ - 배열을 각각 한 번씩만 순회하므로 데이터 규모가 커질수록 압도적으로 유리합니다.

---

## 3. 시행착오 및 해결 과정

### ❌ 정렬 방식의 아쉬움

- 첫 풀이에서는 정렬 후 1:1 비교를 수행했습니다. 코드는 간결하지만 정렬 자체가 가진 시간 복잡도의 한계가 있었습니다.

### ✅ 해시(Map)를 통한 최적화

- **동명이인 처리**: `(pMap.get(name) || 0) + 1` 패턴을 사용하여 `undefined` 방어 로직과 동명이인 카운팅을 한 줄로 해결했습니다.
- **가독성과 성능**: `Map`의 `set`, `get` 메서드를 활용하여 데이터 접근 속도를 $O(1)$로 낮춤으로써 전체 성능을 최적화했습니다.
