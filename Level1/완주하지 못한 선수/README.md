# [Programmers] 완주하지 못한 선수 (Level 1)

## 1. 문제 설명

마라톤에 참여한 선수들의 이름이 담긴 배열 `participant`와 완주한 선수들의 이름이 담긴 배열 `completion`이 주어질 때, 완주하지 못한 단 한 명의 선수의 이름을 반환하는 문제입니다.

### 📋 제한 사항

- 참여 선수의 수: 1명 이상 100,000명 이하
- 완주자 길이는 참가자 길이보다 1 작음
- **동명이인 존재 가능** (가장 주의해야 할 포인트)

---

## 2. 풀이 방식 (정렬 알고리즘 활용)

### 💡 로직 설명

1. **두 배열 정렬**: 참가자(`participant`)와 완주자(`completion`) 배열을 사전순으로 정렬합니다.
2. **순차 비교**: 루프를 돌며 동일한 인덱스에 있는 이름을 비교합니다.
3. **불일치 발견**: 정렬된 상태에서 이름이 다르다는 것은, 해당 인덱스의 참가자가 완주자 명단에 없다는 뜻입니다. (동명이인 문제도 자연스럽게 해결됨)
4. **예외 처리**: 만약 루프가 끝날 때까지 다른 점을 찾지 못했다면, 참가자 배열의 가장 마지막 선수가 미완주자입니다.

---

## 3. 시행착오 및 해결 과정

### ❌ 초기 접근: 반복문과 `filter` (또는 `indexOf`)

- **문제점**: `participant`를 돌면서 `completion`에 이름이 있는지 확인하고 하나씩 지워가는 방식은 최악의 경우 100,000 \* 100,000번의 연산이 필요합니다.
- **결과**: 효율성 테스트에서 '시간 초과'가 발생할 수밖에 없는 구조였습니다.

### ❌ 동명이인 처리의 어려움

- 단순히 `Set` 자료구조를 쓰면 중복된 이름을 가진 사람 중 한 명만 미완주한 경우를 처리할 수 없었습니다. (예: `["mislav", "mislav"]` 중 한 명만 완주했을 때)

### ✅ 최종 해결: 정렬(Sort) 후 1:1 비교

- 두 명단을 똑같이 정렬하면 **"동명이인"들도 나란히 배치**된다는 점을 이용했습니다.
- 이 방식은 한 번의 반복문($O(N)$)으로 비교가 끝나기 때문에 성능과 정확도 면에서 모두 만족스러운 결과를 얻을 수 있었습니다.
